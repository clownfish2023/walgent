View Examples GitHub
image-agent.ts 
import { WalrusAgent, createSkill } from "walgent";

// 1. Define an "image generation" Skill
const imageSkill = createSkill(
  { name: "image-gen", title: "Image Generator", version: "0.1.0" },
  async (ctx) => ({
    ok: true,
    message: `Generated image for "${ctx.input}"`,
    outputs: [{ filename: "cat.svg", contentType: "image/svg+xml", data }],
  }),
);

// 2. Run — the agent auto-uploads the image to Walrus
const agent = new WalrusAgent({ walrus: { network: "testnet" } });
agent.register(imageSkill);

const result = await agent.run("image-gen", { input: "a cat on blockchain" });
console.log(result.stored[0].url);
// → https://aggregator.walrus-testnet.../v1/blobs/CW8p1U4...

Why Walgent

A lightweight TypeScript toolkit that connects AI agents to Walrus storage

🧩

Skill Abstraction

Define agent capabilities with a unified interface, fully decoupling business logic from storage. One createSkill does it all.

🗃️

Auto Upload

After an agent runs, outputs are automatically stored on Walrus — zero boilerplate. Returns decentralized access URLs.

🌐

Multi-Network

Supports mainnet / testnet / devnet with custom endpoints. Full Mainnet private key management built in.

🔁

Retry & Timeout

Built-in exponential backoff retries and request timeout control. Stable uploads even under network fluctuations.

🛡️

Type Safe

Complete TypeScript type definitions. IDE autocompletion and compile-time error checking out of the box.

🧪

Testable

Supports custom fetch mocking for easy unit testing. 25 built-in tests all passing.

How it works

Four steps to decentralized storage for agent outputs

1

Define Skill

Use createSkill to define agent capabilities (image generation, text processing, etc.)

2

Run Skill

WalrusAgent invokes the Skill and produces outputs

3

Auto Upload

Each output is automatically uploaded to the Walrus Publisher

4

Get URL

Returns blobId and decentralized access URL, readable by anyone
